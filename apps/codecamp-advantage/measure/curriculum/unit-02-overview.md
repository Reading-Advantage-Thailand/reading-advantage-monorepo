# Unit 02 Overview: Git & GitHub Fundamentals

**Phase:** A (Foundations)
**Periods:** 4
**Portfolio Project:** Personal Portfolio Website (initial commit)

## Learning Objectives

By the end of this unit, the intern can:

1. Initialize a Git repository (`git init`)
2. Stage, commit, and push changes (`git add`, `git commit`, `git push`)
3. Create and switch between branches (`git branch`, `git checkout`, `git switch`)
4. Fork a repository on GitHub
5. Open a pull request from a fork
6. Create and comment on GitHub Issues
7. Write meaningful commit messages following Conventional Commits
8. Resolve a simple merge conflict

## Technologies & Versions

| Technology | Version | Purpose |
|-----------|---------|---------|
| Git | Latest stable | Version control |
| GitHub | Web | Remote repository hosting |

## Portfolio Connection

The intern creates their Personal Portfolio Website repo on GitHub, makes the initial commit (an `index.html` with their name), and pushes it. This repo will accumulate commits across all Phase A units.

## Key Conventions

- **Commit style:** Conventional Commits (`feat:`, `fix:`, `chore:`, `docs:`)
- **Branch naming:** `feat/add-about-section`, `fix/header-alignment`
- **PR description:** What + Why, not just What

## Prerequisites

- Unit 01 complete (Node.js, pnpm, VS Code installed)
- A GitHub account

## Assessment

- Exercise repo: Fork a practice repo, make changes on a branch, open a PR
- Quiz at the end of Period 4 (5 questions)
