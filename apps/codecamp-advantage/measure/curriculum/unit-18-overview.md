# Unit 18 Overview: Real-World Practice

**Phase:** D (Production)
**Periods:** 4
**Portfolio Project:** Student Progress Tracker (production feature delivery)

## Learning Objectives

By the end of this unit, the intern can:

1. Read and interpret a GitHub Issue with acceptance criteria
2. Implement a feature end-to-end: issue → branch → code → test → PR → review → merge
3. Write a clear PR description that references the Issue
4. Respond to code review feedback professionally
5. Understand the feature delivery lifecycle in a real team

## Technologies & Versions

All technologies from previous units are in play — this is a capstone, not a new technology unit.

## Portfolio Connection

The intern works on pre-filed GitHub Issues on their Student Progress Tracker repo — the same app they've built across Phases C and D. Each Issue describes a real feature or bug fix that makes the tracker more complete.

## Key Concepts

- **Issue-driven development**: Every change starts with an Issue
- **Feature branch workflow**: Create branch → implement → PR → review → merge
- **Code review etiquette**: Kind, constructive, specific feedback
- **Acceptance criteria**: The Issue defines what "done" looks like
- **Small PRs**: Each PR addresses one concern

## Prerequisites

- Units 01–17 complete (all previous units)

## Assessment

- The intern completes 3–4 GitHub Issues as PRs with LLM review
- No quiz — the work itself is the assessment
