# Archived Documentation

This directory stores legacy documentation retained from the pre–git-workflow migration.  
Files here remain available for historical reference but are no longer part of the
spec-first source of truth. Create new specs under `docs/specs/`, update the PRD in
`docs/prd/`, and reference sprint plans in `docs/sprint/`.

## Contents

- `architecture/` – original system architecture diagrams and notes
- `api/` – early API drafts predating the spec migration
- `bug-fixes/` – ad-hoc defect write-ups from legacy process
- `competitor-analysis/` and `competitor-analysis.md` – historical market research
- `curriculum/` – raw curriculum exports captured before spec consolidation
- `front-end-spec/` and `front-end-spec.md` – UI specs from prior workflow
- `onboarding/` – onboarding checklists for the prior AI-assisted workflow
- `github-alignment-summary.md`, `implementation-summary-class-creation-api.md` – point-in-time alignment and implementation summaries
- `science-advantage-project-brief.md` – earlier project brief version kept for context

When migrating information out of this archive, convert it into requirements-focused
specifications or PRD updates so the canonical documents stay current.
