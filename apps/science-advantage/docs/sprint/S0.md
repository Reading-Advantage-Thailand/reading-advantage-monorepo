## Lesson Viewer (Static Content, Completion Toggle)

User Story

- As a student, I want to read Lesson 1 and mark it complete.
- As a teacher, I want to see which students completed Lesson 1.

Acceptance Criteria

- Student opens Lesson 1 and toggles “Mark complete”; state persists per user.
- Teacher sees a completion list per class.

Test Plan

- Unit: lesson viewer renders lesson content; completion toggle updates local state.
- Integration: completion API persists and returns per-user status; teacher list endpoint aggregates by class.
- E2E: student marks complete → teacher page shows student as completed.

Labels: type:feature, area:frontend, area:backend, priority:P1



## MCQ Quiz (Auto-score)

User Story

- As a student, I want to take a 10-question quiz and see my score.
- As a teacher, I want to view per-student scores for a given assignment.

Acceptance Criteria

- Question and attempt models exist; quiz submits and auto-scores.
- Teacher sees scores table for an assignment.

Test Plan

- Unit: scoring function maps answers to score with edge cases (skips, duplicates).
- Integration: submit attempts; verify persisted score and item results; teacher scores endpoint paginates.
- E2E: student completes quiz; score shown; teacher sees row in scores table.

Labels: type:feature, area:backend, area:frontend, priority:P1
