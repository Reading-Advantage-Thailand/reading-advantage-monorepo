## Architecture

```
Next.js apps  →  tRPC  →  domain functions  →  Drizzle  →  Postgres
External HTTP →  Hono   →  domain functions  →  Drizzle  →  Postgres
```

- **tRPC** is the primary product backend interface
- **Hono** is only for external HTTP boundaries (webhooks, health checks)
- **Drizzle** is the database layer (not Prisma — new code uses Drizzle)
- **Domain functions** hold business logic — routers are thin wrappers
- **Multi-tenant** by default — all queries scoped by `schoolId`

## Coding Conventions

### Domain Functions

Every domain function follows this pattern:

```ts
export async function createThing({ db, user, tenant, input }) {
  assertCan(user, "thing:create", tenant)  // permission check first
  const [result] = await db.insert(things).values({ ... }).returning()
  return result
}
```

- Receives `{ db, user, tenant, input }`
- Calls `assertCan()` before any mutation
- Uses `db.transaction()` for multi-row writes
- Returns the created/updated entity

### tRPC Routers

Routers are thin — validate input, enforce auth, call domain functions:

```ts
create: protectedProcedure
  .input(z.object({ ... }))
  .mutation(({ ctx, input }) => createThing({ db: ctx.db, user: ctx.auth.user, ... }))
```

### Multi-Tenancy

Every query must be scoped. Check `user.schoolId` or `tenant.schoolId`. Never trust tenant IDs from the frontend without verifying the user has access.

## Database

- **Local**: Docker Postgres on port 5432, 3 databases (`reading_advantage`, `primary_advantage`, `science_advantage`)
- **Schema**: `packages/db/src/schema/` — organized by domain (users, classrooms, content, progress)
- **Migrations**: `packages/db/drizzle/` — generated by `drizzle-kit generate`
- **Start DB**: `pnpm db:start`

## Build & Test

```bash
pnpm install              # Install dependencies
pnpm db:start             # Start local PostgreSQL
pnpm dev                  # Start all apps
pnpm turbo run lint       # Lint all packages
pnpm turbo run test       # Test all packages
pnpm turbo run build      # Build all packages
pnpm turbo run check-types  # Type check all packages
```

Run for a single package: `pnpm turbo run build --filter=@reading-advantage/db`
